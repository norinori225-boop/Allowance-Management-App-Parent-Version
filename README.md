# おこづかいスタンプ

毎月のおこづかいを「スタンプカード」で管理する、紙の封筒風 PWA アプリです。
子どもの端末がそのまま"スタンプカード"になり、親はQRを読み取ってスタンプを押します。サーバー不要・通信なしで、QRコードのやり取りだけで完結します。

## 構成

| フォルダ | 役割 |
|---------|------|
| `okozukai-child/` | こども側アプリ（カードを表示する端末） |
| `okozukai-parent/` | おや側アプリ（スタンプを押す端末） |

> どちらの `index.html` にも「子どもモード」「親モード」の両方が入っています。用途に合わせて端末ごとに使い分けます。

各フォルダの中身：

| ファイル | 内容 |
|---------|------|
| `index.html` | アプリ本体（HTML/CSS/JS すべて1ファイル） |
| `manifest.webmanifest` | PWA 設定（ホーム画面に追加用） |
| `sw.js` | Service Worker（オフライン動作・キャッシュ） |

## 使い方

1. **こども端末**：「子どもモードをひらく」→ なまえ・毎月のおこづかいを入れて「カードをつくる」。封筒カードとQRが表示されます。
2. **おや端末**：「親モードをひらく」→「QRを読み取る」で子どものカードQLを読み取り、押したい月をタップしてスタンプ → 「子どもに渡す更新QRを表示」。
3. **こども端末**：「📷 親の更新QRを受け取る」で親の更新QRを読み取ると、スタンプが反映されます。

```
こども（カード表示）──QR──▶ おや（スタンプを押す）
       ▲                          │
       └────────更新QR────────────┘
```

## データの保存

- すべて各端末の `localStorage` に保存され、外部に送信されません。
- カード情報キー：`okozukai_child_v1`（なまえ・金額・各月のスタンプ）
- カードの色キー：`okozukai_child_color_v1`

カードのスタンプ情報のみがQRに乗ってやり取りされます（色や端末設定はQRに含めず、端末ごとのローカル設定です）。

## カードのいろを変える（こども側）

こども側の「⚙ 設定・編集」画面から、カードの色を8色から選べます。

| いろ | id |
|------|----|
| クラフト（標準） | `kraft` |
| ピンク | `pink` |
| みずいろ | `blue` |
| みどり | `green` |
| むらさき | `purple` |
| きいろ | `yellow` |
| オレンジ | `orange` |
| ミント | `mint` |

- スウォッチをタップするとその場でプレビュー、「ほぞんする」で確定します。
- 保存せず戻ると元の色に戻ります。
- 仕組み：CSS変数 `--kraft` / `--kraft2` を差し替えて封筒カードとクラフト系ボタンの色を変更しています。

### 色を増やすには

`okozukai-child/index.html` 内の `CARD_COLORS` 配列に1行追加するだけです。

```js
const CARD_COLORS = [
  { id:"kraft",  name:"クラフト", kraft:"#d9c08c", kraft2:"#cbab6e" },
  // ↓ 追加例
  { id:"sky",    name:"そら",     kraft:"#bfe3f2", kraft2:"#8fcbe6" },
];
```

- `id`：保存用のキー（重複しない英字）
- `name`：スウォッチの説明（ツールチップ）
- `kraft`：明るい側の色 / `kraft2`：濃い側の色（カードはこの2色のグラデーション）

## 技術メモ

- 依存ライブラリなし（QR生成・読み取りも含めて `index.html` に同梱）
- カードの内容はQRに `ALW1:` プレフィックス + Base64(JSON) でエンコード
- PWA：`manifest.webmanifest` + `sw.js` によりホーム画面追加・オフライン動作に対応

## 動作確認の注意

ローカルの `index.html` をブラウザで直接開くと、カメラ（QR読み取り）や Service Worker がセキュリティ制限で動かない場合があります。実機テストや配布は HTTPS のホスティング経由で行ってください。
